# 反问策略

在根据不完整旅行输入生成路书前，先使用本参考判断是否需要反问。

## 最低必需信息

如果缺少以下任一信息，必须先追问：

- `origin`：出发城市、地址、车站、机场或当前位置。
- `destination`：目的城市或旅行区域。
- `date_range`：明确日期、相对日期或旅行天数。
- `places_or_interests`：至少一个必去地点或旅行兴趣。

## 可使用的默认值

只有在明确告知用户后，才可以使用默认值：

- `pace`：默认均衡。
- `local_transport`：优先公共交通；时间较晚、地点偏远或行李较多时建议打车。
- `intercity_transport`：用户未指定高铁、飞机或自驾时，默认都可以。
- `lodging`：可以推荐住宿区域，但不要编造具体酒店。

## 提问策略

一次回复最多问 3-5 个问题。

按以下顺序优先提问：

1. 缺失的最低必需信息。
2. 住宿地点或偏好区域。
3. 城际交通偏好。
4. 市内交通偏好。
5. 每日节奏和旅行限制。

不要询问高德 API 可以解决的信息，例如知名地标的精确坐标。

## 必须停止并反问的情况

遇到以下情况时，先停止规划并询问用户：

- 出行日期模糊，且会影响时间安排。
- 出发地或目的地缺失。
- 地点名称在多个城市中都有，存在歧义。
- 用户要求依赖高德 API 不稳定覆盖的数据，例如门票价格、机票价格、酒店价格或精确营业时间。

## 示例

用户：“我想去北京玩三天，去圆明园和环球影城。”

应反问：

“我可以把它做成高德路书。生成前先确认：1. 你从哪里出发？2. 具体出行日期或开始日期是哪天？3. 北京是否已有酒店或偏好住宿区域？4. 城际交通偏好是高铁、飞机，还是都可以？”
