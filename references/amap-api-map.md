# 高德 API 映射

使用本参考判断路书规划中的每一步应使用哪个高德 API。

## 行程理解

- 行政区划查询：规范城市、区县和 adcode。
- 输入提示：在用户补充信息时消解模糊地点名。

## 地点解析

- 地理编码：将地址文本转换为坐标。
- 逆地址编码：将坐标转换为可读地址。
- 关键字搜索：按名称、类型和城市查找 POI。
- 周边搜索：查找附近咖啡馆、书店、餐厅、酒店、车站或备用地点。
- 多边形搜索：在自定义区域内查找 POI，适合按区县或路线走廊规划。
- ID 查询：根据已知 POI id 获取详情。

## 路线规划

- 步行路线：景区内部或相邻地点之间的短距离移动。
- 公共交通路线：用户偏好公共交通时的市内出行。
- 驾车路线：自驾、打车估算、机场接驳或夜间兜底方案。
- 骑行路线：适合短距离城市休闲路线。

对于跨城市高铁和飞机，高德可以辅助解析车站/机场位置和本地接驳路线，但精确班次、票价和余票需要其他数据源或用户自行确认。

## 上下文和风险

- 天气查询：添加每日天气提醒。
- 矩形/圆形/指定线路交通态势：识别自驾或打车路段的拥堵风险。
- 坐标转换：导入第三方坐标时做坐标系规范化。
- IP 定位：只有在用户要求“附近规划”且未提供出发地时，才用于推断默认城市。

## 地图可渲染结果

- 静态地图：生成简单分享图或预览。
- JavaScript API：后续渲染交互式路书。
- 高德 URI 链接：把结果交接到高德 App 或高德网页搜索/导航。

## 不支持或有限支持的事实

除非另有可靠来源，否则不要把以下内容说成来自高德 Web Service 的确认结果：

- 酒店实时价格。
- 景区门票价格。
- 机票价格或精确航班时刻。
- 火车票价格、班次或余票。
- POI 详情未返回的景区营业时间。
